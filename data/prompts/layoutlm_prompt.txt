**INSTRUCTION:**
Извлечь информацию из предоставленного документа счета-фактуры (invoice) и вывести ее в виде **ЕДИНОГО JSON объекта**.
Поля в JSON объекте должны следовать **строго в указанном порядке** и иметь ключи точно так, как они написаны ниже (на русском языке).
Контекст: Компания **{COMPANY_NAME}** является **получателем** счета.

**JSON OUTPUT STRUCTURE AND FIELD EXTRACTION RULES:**

---
- **Поставщик**:
    - **Описание**: Полное наименование компании или ФИО индивидуального предпринимателя, выставившего счет.
    - **Ключевые слова для поиска**: "Поставщик", "Продавец", "Исполнитель", часто в верхней части документа.
    - **Исключение**: Это **НЕ "{COMPANY_NAME}"**.
    - **Формат**: Строка. Точное наименование из документа.
    - **Если не найдено**: "N/A"

- **№ Счета**:
    - **Описание**: Номер счета-фактуры.
    - **Ключевые слова для поиска**: "Счет №", "Invoice #", "№".
    - **Формат**: Строка. **Только цифры/буквы номера**, без префиксов ("№", "б/н") и даты.
    - **Если не найдено**: "N/A"

- **Дата счета**:
    - **Описание**: Дата выставления счета-фактуры.
    - **Ключевые слова для поиска**: "Дата составления", "от", "Date", часто рядом с номером счета.
    - **Формат**: Строка. Строго `DD.MM.YYYY` (например, `15.11.2025`).
    - **Если не найдено**: "N/A"

- **Категория**:
    - **Описание**: Классификация расхода.
    - **Правило**: Выбрать **ОДНУ** из следующих предопределенных категорий на основе анализа товаров/услуг:
        - Инструмент для токарной обработки
        - Инструмент для фрезерной обработки
        - Расходные материалы
        - Прочее
    - **Логика выбора**: Если подходит несколько, выбрать наиболее релевантную. Если определить невозможно, использовать "Прочее".
    - **Формат**: Строка. Точное название категории из списка выше.

- **Товары**:
    - **Описание**: Наименования товаров/услуг из табличной части счета.
    - **Формат**: Строка.
        - Если позиций несколько: перечислить все в одной строке, разделяя знаком **"; "** (точка с запятой и пробел) и переносом строки (каждое наименование на своей строке).
        - Для каждой позиции стремиться к формату: `Наименование - Количество, Общая цена по позиции` (например, `Круг шлифовальный - 10 шт, 5525,00 руб`).
        - Извлекать наименования как есть, без изменений и перевода.
    - **Если не найдено/неприменимо**: "N/A"

- **Сумма без НДС**:
    - **Описание**: Итоговая сумма по счету до вычета НДС.
    - **Ключевые слова для поиска**: "Сумма без НДС", "Итого без НДС", "Net amount", "Subtotal".
    - **Формат**: Строка.
        - Удалить все символы/текст валют (Руб, €, $, EUR, USD, руб., коп.).
        - Использовать **запятую (,) как десятичный разделитель**.
        - Два знака после запятой (например, `"12000,50"`).
    - **Если не найдено**: "N/A"

- **НДС %**:
    - **Описание**: Числовое значение ставки НДС.
    - **Ключевые слова для поиска**: "% НДС", "VAT %", "Ставка НДС".
    - **Формат**: Строка.
        - Использовать **запятую (,) как десятичный разделитель**.
        - **Без знака процента (%)**.
        - Пример: `"20,0"`.
    - **Если НДС не применяется/0%**: `"0,0"`.
    - **Если не найдено**: "N/A" (не пытаться рассчитывать, если не указано явно, т.к. это может быть сложно для LayoutLM без доп. логики).

- **Сумма с НДС**:
    - **Описание**: Итоговая сумма по счету, включая НДС (общая сумма к оплате).
    - **Ключевые слова для поиска**: "Сумма с НДС", "Всего к оплате", "Total amount", "Grand Total".
    - **Формат**: Строка.
        - Удалить все символы/текст валют.
        - Использовать **запятую (,) как десятичный разделитель**.
        - Два знака после запятой (например, `"14400,60"`).
    - **Если не найдено**: "N/A"

- **Валюта**:
    - **Описание**: Трехбуквенный код валюты счета.
    - **Ключевые слова/символы для поиска**: "Валюта", "Currency", символы ₽, €, $.
    - **Формат**: Строка. Верхний регистр (например, `"RUB"`, `"EUR"`, `"USD"`).
    - **Если не найдено**: "N/A"

- **Комментарии**:
    - **Описание**: Дополнительные релевантные примечания (условия оплаты, поставки и т.д.).
    - **Ключевые слова для поиска**: "Примечание", "Условия", "Комментарий".
    - **Формат**: Строка.
    - **Если не найдено/неприменимо**: "N/A"

---

**ОБЯЗАТЕЛЬНЫЕ ТРЕБОВАНИЯ К ВЫВОДУ:**
1.  **Вывод**: ИСКЛЮЧИТЕЛЬНО JSON объект. Без дополнительного текста.
2.  **Структура и порядок**: Строго соблюдать указанный порядок полей и ключи на русском языке.
3.  **Значения**: ВСЕ значения в JSON должны быть строками (в кавычках).
4.  **Форматирование**: Неукоснительно соблюдать форматы дат, чисел и валют, как описано для каждого поля.
5.  **Язык**: Извлекать текст **точно как в документе**. НЕ ПЕРЕВОДИТЬ. Сохранять оригинальную орфографию.
